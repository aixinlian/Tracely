### 3. ✅ 生成日报页面添加复制按钮和重新生成功能
**文件**: `src/pages/Daily.tsx`, `src/components/report-generation-provider.tsx`

**新增功能**：
- 在日报内容右上角添加"复制"按钮
- 点击后复制日报内容到剪贴板
- 复制成功后显示"已复制"状态，2秒后恢复
- **重新生成功能**：当检测到日报是从历史记录加载时，显示"重新生成"按钮
  - 允许用户强制重新生成最新的日报内容
  - 重新生成会更新现有的历史记录，而不是创建新记录
  - 提示文字变为"已从历史记录加载，点击重新生成可获取最新内容"

**技术实现**：
- 导入 `Copy`、`Check` 和 `RefreshCw` 图标
- 添加 `copied` 状态管理
- 实现 `handleCopy()` 函数，使用 `navigator.clipboard.writeText()`
- 在 context 中添加 `isFromCache` 状态标记
- `generate()` 函数支持 `forceRegenerate` 参数
- 强制重新生成时调用 `updateReport()` 更新现有记录

---

### 4. ✅ 历史记录页面添加全选功能
**文件**: `src/pages/History.tsx`

**新增功能**：
- 每个报告卡片左侧添加复选框
- 列表上方添加"全选/取消全选"按钮
- 选中报告后显示"删除选中 (N)"按钮

**技术实现**：

- 导入 `CheckSquare` 和 `Square` 图标
- 实现以下函数：
  - `toggleSelectAll()`: 全选/取消全选
  - `toggleSelect(id)`: 切换单个报告选中状态
  - `handleDeleteSelected()`: 批量删除选中报告
- 修改 `ReportCard` 组件，添加 `selected` 和 `onToggleSelect` 属性
- 删除单个报告时同步更新 `selectedIds`

2. 验证默认主题为浅色模式
3. 切换到深色模式，刷新页面，验证设置被保存

### 测试日报生成
1. 添加至少一个项目
2. 配置 AI 供应商
3. 生成日报，检查格式是否为列表式（1. 2. 3. ...）
4. 验证每条工作描述是否清晰、具体

### 测试复制按钮
1. 生成日报后，点击右上角"复制"按钮
2. 验证按钮显示"已复制"状态
3. 粘贴到文本编辑器，确认内容正确
4. 等待2秒，验证按钮恢复为"复制"

### 测试重新生成功能
1. 生成一份日报（例如今天的日报）
2. 提交一些新的代码或做一些改动
3. 再次进入日报页面，点击"生成日报"
4. 验证显示的是之前的日报内容，且提示为"已从历史记录加载"
5. 验证右上角出现"重新生成"按钮
6. 点击"重新生成"按钮
7. 验证生成了包含最新改动的日报
8. 检查历史记录页面，确认没有重复的报告（只更新了原有记录）

### 测试历史记录全选
1. 确保有多条历史报告
2. 点击"全选"按钮，验证所有报告被选中
3. 手动取消部分选中，验证计数正确
4. 点击"删除选中"按钮，验证批量删除成功
5. 删除单个报告时，验证选中状态同步更新

### 6. ✅ 清除缓存功能
**文件**: `src/pages/Settings.tsx`, `src/lib/settings.ts`

**功能**：
- 替换原有的"数据目录"功能
- 添加"清除所有数据"按钮，带有红色警告样式
- 点击后弹出确认对话框，详细列出将要删除的内容
- 清除所有 IndexedDB 数据（项目、供应商、报告）
- 清除所有 localStorage 设置（主题、Git 用户名等）
- 清除完成后自动刷新页面

**技术实现**：
- 新增 `clearAllData()` 函数
- 使用 `db.delete()` 清除 Dexie 数据库
- 清除 localStorage 中的所有应用相关键值
- 带有二次确认对话框，防止误操作
- 清除过程中显示加载状态

**原因**：
数据目录功能因为数据存储在 IndexedDB 中而显得多余，用户更需要清除缓存功能来重置应用状态。
